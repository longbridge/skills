# 数据获取规范

## 概述

本文件定义 Skill 运行时的数据获取策略，包括需要哪些数据、从哪里取、取不到怎么办。

## Longbridge CLI

数据通过 `longbridge` 命令行工具获取。CLI 持续迭代中，**不要假设任何具体参数**。

执行任何命令前，必须先通过 `--help` 确认当前支持的子命令和参数：

```bash
longbridge --help                          # 顶层命令列表
longbridge <command> --help                # 子命令与选项
longbridge <command> <subcommand> --help   # 更深一层
```

先看 `--help`，再执行命令。不要跳过这一步。要严格按照 help 中的指令来，不要做多余的动作，当报错的时候重新看一下支持的命令格式。

**禁止编造参数。** 只使用 `--help` 输出中明确列出的参数。如果 `--help` 里没有某个参数（如 `--market`、`--start`、`--end`、`--symbol` 等），就不要使用，即使觉得「应该有」也不行。

**先拉数据，再筛选。** 用最简单的方式拉取数据（通常不带或只带最少的参数），拿到结果后再根据用户需求（时间、市场、重要性等）从结果中筛选。所有过滤逻辑都在拿到数据之后进行。

## 需要的数据

### 用户数据（不可降级）

| 数据 | 用途 | CLI 命令方向 |
| --- | --- | --- |
| 持仓列表 | 确定用户关注的标的和市场 | `longbridge positions` |
| 自选股列表 | 同上 | `longbridge watchlist` |

两者均需获取，缺一不可。持仓和自选股共同构成用户关注的完整标的范围，任一获取失败时终止个性化流程，提示用户检查登录状态（`longbridge auth login`）。

### 财经日历数据（可降级到 WebSearch）

| 数据 | CLI 命令方向 |
| --- | --- |
| 财报/业绩日历 | `longbridge finance-calendar` 下的相关子命令 |
| 宏观经济日历 | 同上 |
| 分红/除权日历 | 同上 |
| 休市/交易日历 | 同上 |
| 拆合股日历 | 同上 |

`finance-calendar` 有多个子命令（如 report、macrodata、dividend、closed、split 等），具体用法和支持的过滤参数请通过 `longbridge finance-calendar --help` 和各子命令的 `--help` 确认。

关键提示：财报日历的返回数据中通常已包含 EPS 预期值和实际值，不需要额外调用其他命令获取。

### 股票代码格式

Longbridge 使用 `<CODE>.<MARKET>` 格式（如 `AAPL.US`、`700.HK`、`600519.SH`）。具体支持的市场后缀见 `longbridge --help` 顶部说明。

## 降级规则

1. **用户数据不可降级**：持仓和自选股必须通过 CLI 获取。未授权时仅提供全局财经日历查询。
2. **日历数据可降级**：CLI 调用失败或返回空结果时，使用 WebSearch 补充，优先搜索权威财经网站（交易所公告、公司 IR 页面、主流财经媒体）。
3. 降级时无需告知用户数据来源差异。
