# 用户意图分类

用户通过 CLI 与 skill 交互，输入自然语言指令。skill 需要将指令分类为以下 8 种意图之一，然后执行对应操作。**如果无法确定意图，默认为"查看晨报"（意图1）。**

## 意图判断优先级

当指令可能匹配多个意图时，按以下优先级判断:

1. 包含明确的设置动词（"改"/"调"/"设置"/"关掉"）→ **意图6**
2. 包含"加到自选"/"删掉" → **意图5**
3. 包含"全景"/"完整"/"profile"/"档案" → **意图4**
4. 包含时间范围 + "回顾"/"历史" → **意图7**
5. 包含跨市场关联词 → **意图8**
6. 包含具体股票名但无上述关键词 → **意图3**
7. 包含市场名但无具体股票 → **意图2**
8. 其余所有情况 → **意图1（默认晨报）**

## 意图1: 查看晨报

**描述**: 用户想要查看当前所有市场的催化剂增量信号汇总。最高频的意图。

**触发词**:
- 中文: "今天有什么要关注的"、"晨报"、"早报"、"晚报"、"盘前看看"、"有什么消息"、"今天有什么信号"、"催化剂雷达"、"帮我扫描一下"
- 英文: "morning briefing"、"what should I watch today"、"any signals"、"catalyst update"、"daily brief"

**操作**:
1. 判断各市场当前状态（Longbridge Trading Session API）
2. 确定输出早报还是晚报（基于距开盘/收盘的时间）
3. 获取自选股列表（Longbridge Watchlist API）
4. 按优先级分层扫描（longbridge-api-map batch_strategy）
5. 信号处理
6. 输出完整晨报（briefing-templates 模板一）

## 意图2: 查看特定市场

**描述**: 用户只想看某个市场的信号。

**触发词**: "美股昨晚怎么样"、"A股有什么信号"、"港股早报"、"看看新加坡"、"US market update"、"HK briefing"

**意图识别逻辑**: 从指令中提取市场关键词 → 映射到市场代码
- "美股" / "US" / "纳斯达克" / "标普" → US
- "A股" / "沪深" / "上证" / "深圳" → CN
- "港股" / "HK" / "恒生" / "恒指" → HK
- "新加坡" / "SG" / "海峡时报" → SG

**操作**: 与意图1相同，但只输出指定市场（briefing-templates 模板二）

## 意图3: 查看特定股票

**描述**: 用户想看某只股票近期的催化剂信号。

**触发词**: "NVDA最近有什么催化剂"、"茅台有什么消息"、"看看腾讯"、"0700.HK怎么样"、"what's happening with AAPL"

**意图识别逻辑**: 从指令中提取股票标识 → 匹配到具体代码
- 代码: "NVDA"、"600519"、"0700.HK" → 直接匹配
- 公司名: "茅台" → 600519.SH、"腾讯" → 0700.HK、"英伟达" → NVDA.US（从 Longbridge Security Static API 获取映射）

**操作**:
1. 对该股票做 full_scan（不受批量扫描分层限制）
2. 拉取最近 7 天的信号（包括冷却中的历史信号，但标注"已推送"）
3. 按时间倒序输出
4. 如果 7 天内无信号，告知并提供最近一次信号的时间

**输出格式（非标准模板，临时格式）**:
```markdown
## {stock_name}({code}) 近7天催化剂信号
> {market_flag} {price} ({change_pct}) | 最后扫描: {now}

{日期} [{level}] {signal_summary}
{日期} [{level}] {signal_summary}
...

> 如需完整分析，请输入"全景扫描 {stock}"
```

## 意图4: 查看全景档案

**描述**: 用户想看一只股票的 7 维完整分析，不受增量过滤限制。

**触发词**: "给我看NVDA的完整分析"、"全景扫描茅台"、"NVDA profile"、"腾讯档案卡"、"full analysis"、"complete scan"

**操作**:
1. 检查档案卡缓存
   - 缓存有效（< 24小时且无强制刷新条件）→ 使用缓存 + 刷新价格
   - 缓存过期或无缓存 → 全量调用 API（约 10-15 calls）
2. 按档案卡模板输出
3. 更新缓存


---

## 意图5: 调整设置

**描述**: 用户想调整推送模式、灵敏度、维度开关等偏好。

**触发词**: "A股只看早报就行"、"灵敏度调高一点"、"美股晚报改到5点推"、"关掉技术面信号"、"只看财报和内部人"

**操作**: 更新对应字段:
| 用户说 | 更新字段 |
|--------|---------|
| "只看早报" | `preferences.markets.{market}.push_mode = "once_daily"` |
| "灵敏度调高" | `preferences.sensitivity = "high"` |
| "关掉技术面" | `preferences.dimensions_enabled` 中移除 "technical" |
| "晚报改到5点" | `preferences.markets.{market}.evening_offset_min = 相应值` |
| " 只看A股" | `preferences.markets.{market}.enabled`更新用户偏好 


操作后确认并说明影响: "已将A股改为仅早报模式，每日08:30推送"

## 意图6: 回溯查看

**描述**: 用户想看某只股票或某个时间段的历史信号。

**触发词**: "上周NVDA有哪些信号"、"这个月茅台的催化剂时间线"、"回顾一下上周"、"过去30天港股有什么大事"

**时间范围解析**:
- "上周" → 过去7天
- "这个月" → 月初至今
- "过去30天" → 30天

**操作**: 从历史记录中检索匹配记录，按时间正序排列，输出时间线

**输出格式**:
```markdown
## {stock_name} 催化剂时间线 | {start_date} — {end_date}

**{date}** [{level}] {signal_summary}
**{date}** [{level}] {signal_summary}
**{date}** [{level}] {signal_summary}

> 共 {count} 条信号 | 重大 {critical_count} | 关注 {watch_count} | 常规 {info_count}
```

## 意图7: 跨市场联动

**描述**: 当一个市场的股票事件对另一个市场的影响时触发。

**触发情况**: "美股半导体大涨，A股半导体产业链同样受益"、"芯片出口管制对中美半导体的影响"（包含两个市场的引用，或包含"联动"/"影响"/"传导"）

**操作**:
1. 识别源市场/源事件
2. 查询跨市场关联表找到关联股票
3. 对关联股票快速扫描（quote_only）
4. 输出联动分析（briefing-templates 模板四）
