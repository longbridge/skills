# 持仓驱动财经日历简报

本文件定义默认工作流：以用户持仓 + 自选股为中心，生成「事件总览 → 重点解读 → 财报速递」三段式报告。

## 前提

用户已完成 Longbridge 账户授权。未授权时的降级策略见 `references/data-fetching.md`。

## 数据驱动，而非预配置

所需信息从以下来源获取，不维护独立的用户偏好配置：

- **持仓与自选股**：从 Longbridge 账户实时获取，自动确定用户关注的标的和市场范围。**持仓与自选股在事件覆盖上同等重要，均需完整扫描，不可省略任一来源。** 持仓和自选不分组，统一按事件时间排序，仅用 tag 区分来源
- **时间范围**：从用户请求判断，默认规则如下。CLI 不一定支持时间参数，应先拉取数据再从结果中按时间筛选

| 用户表述 | 扫描范围 |
| --- | --- |
| 今天 / 今日 | 当日（T+0） |
| 明天 | T+1 |
| 这周 / 本周 | 本周一至周日 |
| 下周 | 下周一至周日 |
| 最近 / 近期 | T+0 ~ T+3（未来 3 个自然日） |
| 未明确说明时间 | 默认 T+0 ~ T+3 |

**跨周末与非交易日规则：**

时间范围以自然日计算，但必须确保覆盖到下一个交易日：

1. **自动延展**：当时间范围的末尾落在周末或节假日时，自动延展到该市场的下一个交易日
2. **多市场分别判断**：用户持仓跨多个市场时（如美股+港股+A股），每个市场的交易日历不同，需分别判断
3. **利用休市日历数据**：数据采集阶段已拉取休市日历，用它来确定每个市场的实际交易日边界
4. **输出中标注**：当时间窗口因跨周末/假日而延展时，在标题中体现实际覆盖范围（如「5月8日至5月11日」而非「未来3天」）

- **输出风格**：默认使用常识性语言（日常用语，不含专业术语）；若用户表现出专业背景或主动要求，切换为基本面风格（保留原始数据）
- **过滤与优先级**：优先展示高重要性事件和与用户持仓直接相关的事件。**事件范围不局限于用户持仓和自选股**：全局重大事件（如美联储议息、非农数据等）和与用户持仓/自选所在行业关联的热点事件也应纳入

上述默认行为无需用户预先配置。若用户在对话中提出调整，在当次对话中遵照执行即可。

## 执行流程

每次请求均执行完整的三步流程，输出包含三个部分。每个部分有数据就生成，无数据则省略该部分（但第一部分必须保留）。尽量多生成内容，覆盖用户能关注到的所有维度。同一事件只出现一次，不在不同部分重复描述。

### 数据采集（统一前置）

1. 获取用户持仓与自选股列表（未授权时终止并提示登录）
2. 按 `data-fetching.md` 规范，**并行拉取所有数据源**：
   - 财报/业绩日历
   - 宏观经济日历
   - 分红/除权日历
   - 休市/交易日历
   - 拆合股日历
3. 从返回结果中按用户请求的时间范围筛选（默认 T+0 ~ T+3）
4. **时间方向：聚焦未来。** 只保留**尚未发生**的事件（今日起往后）。唯一例外是「昨夜/今日盘前」刚出炉的财报结果（即上一个交易日收盘后至今日开盘前发布的），可在第一部分和第三部分简要提及
5. 若用户指定了特定股票，则在该股票维度做深度展开，但其他部分仍覆盖全量持仓与自选

### 第一部分：事件总览（对应 output-template.md 模板一）

所有事件（宏观、持仓、自选）混合在一条时间线上，按日期和时间统一排序，不重复：

- 若有**昨夜或今日盘前**刚出炉的财报结果，简要提及，不展开历史
- 按自然日分组，同一天内按时间先后排列
- 宏观事件和标的事件混排在同一时间线中，不单独分组
- 每个标的后用「持仓」或「自选」tag 标识来源
- 无事件的标的不出现；事件数量不设上限

### 第二部分：重点事件影响解读（对应 output-template.md 模板二）

- 从第一部分中提取高重要性事件（如重磅财报、美联储议息、非农数据、行业重大政策等）
- 对每个重点事件做深度解读：事件内容与核心数据、对持仓和自选股的影响分级（直接影响 / 间接影响）、行动参考（不含买卖指令）
- 若时间范围内无高重要性事件，此部分可省略

### 第三部分：财报结果速递（对应 output-template.md 模板三）

- 仅覆盖**昨夜或今日盘前**刚出炉的财报结果，逐一生成财报结果卡片
- 若无已出炉财报，此部分省略

### 收尾

- 末尾附加免责文案（见输出规范）
- 提示用户可追问：「有任何事件想深入了解，或者财报结果出来后，随时问我」

## 输出规范

### 语言规则

- 句子长度 ≤ 30 字，避免长从句
- 每条事件结论不超过 2 行
- 使用「可能」「大概率」等表述，不给确定性预测
- 行动建议用「建议观察」「等开盘后再看」，不用「应该买/卖」
- 不出现 EPS、guidance、Beta、IV 等专业术语，统一转化为日常语言

### 术语转化对照

| 专业术语 | 输出用语 |
| --- | --- |
| 除权除息日（Ex-Date） | 注意区分：不同市场享受分红的规定差异，这天之后买入就拿不到这次分红了 |
| 半日市 | 港股今天只开到中午 |
| 产业链传导 | 这家公司的消息可能带动相关股票一起涨跌 |
| 流动性风险 | 那天市场交投清淡，价格波动可能比较大 |
| 宏观数据超预期 | 经济数据比市场预测的好 |

### 禁止项

- 不回复「今日无重要内容」的空结果
- 不给具体价格预测
- 不直接建议买入或卖出
- 自选股有事件时必须展示；无事件的标的直接省略
- 全局重大事件和与用户持仓/自选所在行业直接关联的热点标的允许出现；其余情况不推荐未持有且未自选的股票

### 免责文案

每份完整报告末尾固定附加一次：

> 有任何事件想深入了解，或者财报结果出来后，随时问我。
> 以上内容基于longbridge数据源和公开信息整理，仅供参考，不构成投资建议，请结合自身判断决策。
