## PAGES 8-10: VALUATION & ESTIMATES

### Updated Valuation (1-2 pages)

**IMPORTANT: Use actual calculations with real data from longbridge CLI — NOT placeholder templates.**

Use `longbridge calc-index --help`, `longbridge static --help`, `longbridge kline --help`, and `longbridge filing --help` to fetch market and financial data for subject and peer companies.

**See [valuation-methodologies.md](valuation-methodologies.md) for complete calculation details, formulas, and pitfalls.**

**Method 1: DCF Analysis (weight 40-60%)**

8-step process: Historical analysis → Revenue projections → OpEx → FCF → Terminal Value → WACC → Discount → Equity bridge. Run under Bear/Base/Bull scenarios (varying growth, WACC ±50bps, terminal growth).

**Method 2: Trading Comparables (weight 25-40%)**

Select 5-10 peers → calculate EV/EBITDA, P/E, EV/Revenue → use median → apply premium/discount. Use 25th percentile (Bear), median (Base), 75th percentile (Bull).

**Method 3: Precedent Transactions (weight 0-25%, only if M&A data available)**

Longbridge CLI does not provide M&A data. Use web search for recent industry transactions (last 3-5 years). Calculate transaction EV/EBITDA and control premiums. Only include this method when relevant transactions exist.

If no precedent data available, redistribute weight to DCF and Comps.

### Dynamic Weighting

**Do NOT use fixed weights.** Adjust based on context:

| Factor | Higher DCF Weight | Higher Comps Weight | Include Precedent |
|--------|-------------------|--------------------|--------------------|
| Forecast confidence | High → 50-60% | Low → 35-40% | — |
| Market conditions | Abnormal/dislocated | Normal/stable | — |
| Company maturity | Growth stage | Mature/stable | — |
| M&A likelihood | — | — | Industry consolidating → 15-25% |

State your chosen weights and rationale in the report.

### Bear/Base/Bull Valuation Range

**Always present a valuation RANGE, not a single point estimate.**

```
VALUATION SUMMARY
─────────────────────────────────────────────────────────────
                         Bear Case    Base Case    Bull Case
DCF (XX% weight)         $XXX         $XXX         $XXX
Trading Comps (XX%)      $XXX         $XXX         $XXX
Precedent Trans (XX%)    $XXX         $XXX         $XXX
─────────────────────────────────────────────────────────────
Weighted Average          $XXX         $XXX         $XXX

Price Target: $XXX (Base Case weighted average)
Prior Target: $XXX
Current Price: $XXX (from longbridge quote)
Implied Upside: +XX%
```

**Scenario assumptions:**
- **Bear**: Conservative growth, WACC +50bps, 25th percentile peer multiples
- **Base**: Most likely estimates updated from Q[X] results
- **Bull**: Optimistic growth, WACC -50bps, 75th percentile peer multiples

### Sanity Checks
- DCF implied price within ±30% of current market price — if not, explain divergence
- All methods should directionally agree — major outlier needs explanation
- Terminal value = 50-70% of total EV (if >75%, re-examine assumptions)
- Historical multiples: is current valuation in line with company's own history?
- Implied growth: what growth rate is the market pricing in? Is it realistic?

## FORMATTING REQUIREMENTS

### 1. Page 1 Requirements
- Clear rating (MAINTAIN OUTPERFORM, RAISE TO BUY, etc.)
- Updated price target prominently displayed
- Summary table with old/new estimates
- 3-4 paragraph-length bullets with ■ character

### 2. All Tables Requirements
- Source line at bottom
- Clear column headers
- Shading for header rows

### 3. All Charts Requirements
- "Figure X - [Title]" caption above
- "Source: [Source]" line below
- Professional styling

### 4. Year Notation
- Use A for actual (Q3'24A)
- Use E for estimate (Q4'24E)

### 5. Writing Style
- Lead with numbers ("Revenue grew 15% to $1.2B" not "Strong revenue growth")
- Use "vs." not "versus"
- Be direct and concise
- Focus on what's NEW

### 6. Hyperlink Requirements ⭐⭐⭐
- ALL URLs must be clickable hyperlinks in Word
- Blue, underlined text that opens on Ctrl+Click
- Display text meaningful (not raw URL)
- Every source citation should have clickable link where applicable
- No plain text URLs - always format as hyperlinks
